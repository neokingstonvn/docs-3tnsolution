3. Database Schema (PostgreSQL DDL SQL)

Tên bảng Thực thể chính Mục tiêu lưu vết nghiệp vụ chi tiết
users Nhân viên & Vai trò Lưu ID (UUID), Email, Họ tên, Role (Admin/PM/Staff) để phân quyền.
projects Dự án văn phòng Lưu ID, Tên dự án, PM ID (Khóa ngoại tham chiếu users.id), Trạng thái (Active/On Hold/Completed).
tasks Công việc chi tiết Lưu ID, Project ID, Tiêu đề, Assignee ID, Priority, Due date, Trạng thái (To Do/In Progress/Reviewing/Done).
comments Bình luận thảo luận Lưu ID, Task ID, User ID, Nội dung bình luận, Thời gian.
approval_requests Yêu cầu phê duyệt Lưu ID, Task ID, Submitter ID, Báo cáo, Link tài liệu, Approver ID, Trạng thái (Pending/Approved/Rejected), Phản hồi.
activity_logs Lịch sử hoạt động Lưu ID, Task ID, User ID, Mô tả hành động (Action), Thời gian. 
-- BẢNG USERS
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    full_name VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL CHECK (role IN ('Admin', 'Project Manager', 'Staff')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- BẢNG PROJECTS
CREATE TABLE projects (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    description TEXT,
    pm_id UUID REFERENCES users(id) ON DELETE SET NULL,
    status VARCHAR(50) NOT NULL DEFAULT 'Active' CHECK (status IN ('Active', 'On Hold', 'Completed')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- BẢNG TASKS
CREATE TABLE tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID REFERENCES projects(id) ON DELETE CASCADE NOT NULL,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    assignee_id UUID REFERENCES users(id) ON DELETE SET NULL,
    priority VARCHAR(20) NOT NULL CHECK (priority IN ('High', 'Medium', 'Low')),
    due_date DATE NOT NULL,
    status VARCHAR(50) NOT NULL DEFAULT 'To Do' CHECK (status IN ('To Do', 'In Progress', 'Reviewing', 'Done')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- BẢNG COMMENTS
CREATE TABLE comments (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id UUID REFERENCES tasks(id) ON DELETE CASCADE NOT NULL,
    user_id UUID REFERENCES users(id) ON DELETE CASCADE NOT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- BẢNG APPROVAL_REQUESTS
CREATE TABLE approval_requests (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id UUID REFERENCES tasks(id) ON DELETE CASCADE NOT NULL,
    submitter_id UUID REFERENCES users(id) ON DELETE CASCADE NOT NULL,
    report_text TEXT NOT NULL,
    report_link VARCHAR(512),
    approver_id UUID REFERENCES users(id) ON DELETE SET NULL,
    status VARCHAR(50) NOT NULL DEFAULT 'Pending' CHECK (status IN ('Pending', 'Approved', 'Rejected')),
    feedback TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- BẢNG ACTIVITY_LOGS
CREATE TABLE activity_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id UUID REFERENCES tasks(id) ON DELETE CASCADE NOT NULL,
    user_id UUID REFERENCES users(id) ON DELETE CASCADE NOT NULL,
    action TEXT NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

4. Danh Sách Task Triển Khai Chi Tiết

Dưới đây là sơ đồ chia nhỏ công việc thực thi MVP. Bạn có thể tích chọn trực tiếp các hộp kiểm dưới đây để theo dõi và cập nhật tiến độ tự động (trạng thái tiến độ sẽ được lưu lại trên trình duyệt của bạn).

Giai Đoạn 1 Cấu Hình Core Database & Module Auth (Ngày 1–3)

  • Task 1.1: Thiết lập database migration & DDL SQL Schema PostgreSQL

    Tạo các bảng users, projects, tasks, comments, approval_requests, activity_logs và các indexes trên PostgreSQL.

  • Task 1.2: Scaffold ứng dụng Node.js/Express theo Clean Architecture Node.js

    Cài đặt express, pg-pool, dotenv, cors và thiết lập cấu trúc 3 lớp: Controllers, Services, và Repositories.

  • Task 1.3: Viết middleware RBAC kiểm tra quyền truy cập theo vai trò Node.js

    Hiện thực middleware JWT/Guards giải mã token và kiểm tra quyền RBAC của người dùng trước khi vào router.

Giai Đoạn 2 APIs & Giao Diện Quản Lý Dự Án (Ngày 4–6)

  • Task 2.1: Viết REST APIs CRUD quản lý Dự án Node.js

    Xây dựng endpoints POST /api/projects, GET /api/projects, và PATCH /api/projects/:id.

  • Task 2.2: Thiết lập Giao diện Dashboard chính & Sidebar Navigation Vanilla JS

    Khởi tạo bố cục khung app, thanh tiêu đề, menu định tuyến và xử lý chuyển đổi giao diện mượt mà.

  • Task 2.3: Màn hình danh mục dự án (Project Directory) & Form Tạo mới Vanilla JS

    Hiển thị danh sách dự án dưới dạng thẻ card kèm hiển thị phần trăm tiến độ tự động; tích hợp form modal thêm mới dự án.

Giai Đoạn 3 Kanban Board & APIs Công Việc (Ngày 7–10)

  • Task 3.1: Viết REST APIs CRUD quản lý Công việc Node.js

    Hiện thực các endpoints: POST /api/tasks, GET /api/projects/:id/tasks, PATCH /api/tasks/:id/status.

  • Task 3.2: Giao diện Kanban Board kéo thả cập nhật tiến độ trực quan Vanilla JS

    Thiết kế 4 cột trạng thái: To Do, In Progress, Reviewing, Done kèm tương tác kéo thả thẻ HTML5 Drag and Drop.

  • Task 3.3: Giao diện xem chi tiết Task, Comment Box & Activity Logs Vanilla JS

    Tích hợp modal hiển thị mô tả nhiệm vụ, danh sách lịch sử sửa đổi trạng thái và khung bình luận trực tiếp.

Giai Đoạn 4 Log Engine & Trình Phê Duyệt Trạng Thái (Ngày 11–13)

  • Task 4.1: Log Engine tự động ghi nhận thay đổi trạng thái Node.js

    Thiết lập hook hoặc trigger trung gian tự động chèn bản ghi vào bảng activity_logs khi cập nhật task.

  • Task 4.2: APIs đệ trình và phê duyệt hoàn thành nhiệm vụ Node.js

    Endpoints POST /api/tasks/:id/submit-approval và POST /api/approvals/:id/review để duyệt/từ chối.

  • Task 4.3: Giao diện hàng đợi phê duyệt (Approval Queue) dành cho PM/Admin Vanilla JS

    Màn hình gom nhóm các nhiệm vụ đang chờ phê duyệt, hiển thị báo cáo, đính kèm link tài liệu và form nhập feedback từ chối.

Giai Đoạn 5 Kiểm Thử UAT & Mock Database (Ngày 14–15)

  • Task 5.1: Xây dựng bộ kịch bản kiểm thử Happy Path & E2E QA

    Kiểm tra toàn bộ vòng đời của một dự án và công việc từ lúc tạo cho tới khi hoàn thành.

  • Task 5.2: Triển khai CSDL giả lập LocalStorage vào file db.js Vanilla JS

    Liên kết database mock với các view HTML để đảm bảo Prototype UAT hoạt động độc lập không cần server backend.

  • Task 5.3: Đóng gói và viết hướng dẫn triển khai sản xuất DevOps

    Viết Docker Compose cấu hình Node.js/PostgreSQL và tài liệu hướng dẫn chạy lệnh seed dữ liệu thật.

5. Đồng Bộ REST API & Stack Kỹ Thuật

Chi tiết API Router:

  • POST /api/projects - Tạo mới dự án (Quyền Admin)
  • GET /api/projects - Danh sách dự án (Tất cả vai trò, lọc theo quyền)
  • POST /api/tasks - Tạo task mới (PM phụ trách, Admin)
  • POST /api/tasks/:id/submit-approval - Gửi yêu cầu duyệt (Staff)
  • POST /api/approvals/:id/review - Duyệt/Từ chối task (PM, Admin)
