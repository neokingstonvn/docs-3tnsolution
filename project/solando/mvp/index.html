4. Đặc Tả Tính Năng MVP

🔑
Module 1

Xác Thực & Phân Quyền (Auth & RBAC)

⚙️ System Features

  • Đăng nhập Email + Password, mã hóa Bcrypt
  • Cấp JWT Token kèm thông tin Role
  • RBAC: Chặn truy cập API theo Role ở tầng middleware
  • Ghi nhận Audit Log (ai, làm gì, lúc nào)

👤 User Features

  • Trang đăng nhập chung cho nhân viên nội bộ
  • Admin: Tạo, sửa, khóa tài khoản nhân viên + gán Role
  • Trang đổi mật khẩu cá nhân
📦
Module 2

Quản Lý Sản Phẩm (SKU)

📐 Business Rules

  • Mỗi SKU có: Tên, Mã SKU, Hãng, Nhóm (Inverter / Pin / Tấm pin / Phụ kiện), Thời hạn bảo hành (tháng), MSRP
  • SKU gắn liền với file tài liệu: Datasheet, CO/CQ, Catalog
  • Thiết bị chính (Inverter, Pin Lithium): bắt buộc quản lý Serial Number. Phụ kiện: quản lý theo số lượng
  • Thời hạn bảo hành mặc định cấu hình trên từng SKU (ví dụ: Inverter 60 tháng, Pin Lithium 120 tháng)

⚙️ System Features

  • Phân loại sản phẩm tự động theo nhóm
  • Lưu trữ và quản lý phiên bản file tài liệu kỹ thuật

👤 User Features (Admin / Sale)

  • Danh sách SKU: Tìm kiếm, lọc theo nhóm, hãng
  • Form tạo/sửa SKU: thông số kỹ thuật, upload CO/CQ, Datasheet
  • Cấu hình bảng giá theo Tier (Tier 1 / Tier 2 / Bán lẻ) cho từng SKU
🏢
Module 3

Quản Lý Đại Lý & Khách Hàng

📐 Business Rules

  • Đại lý được phân hạng: Tier 1, Tier 2, Khách lẻ
  • Mỗi đại lý được gán cho một nhân viên Kinh doanh (Sale) phụ trách chăm sóc
  • End-User lưu kèm Đại lý phụ trách (liên kết qua kích hoạt QR)

⚙️ System Features

  • Phân loại và tự động phân bổ thông tin đại lý theo khu vực hoặc sale phụ trách

👤 User Features (Sale / Admin)

  • Danh sách Đại lý: Tier, Sale phụ trách, thông tin liên hệ
  • Form tạo/sửa Đại lý: Tên, MST, Địa chỉ, SĐT, Email, Tier
  • Chi tiết Đại lý: Lịch sử đơn hàng, thông tin liên hệ chi tiết
  • Danh sách End-User: Tra cứu theo tên, SĐT hoặc Đại lý
📋
Module 4

Báo Giá & Pricing Engine

📐 Business Rules

  • Khi Sale chọn Đại lý → hệ thống tự áp bảng giá theo Tier
  • Sale có thể nhập thêm chiết khấu % trên từng dòng sản phẩm
  • Margin % tính dựa trên giá vốn lô hàng gần nhất đang có trong kho
  • Margin dưới ngưỡng → Báo giá bị khóa "Chờ duyệt" → Admin nhận In-App notification
  • ⚠ Ngưỡng Margin cụ thể sẽ cấu hình trong System Config — chưa xác định
  • Luồng: Báo giá → Đại lý đồng ý miệng → Sale chuyển thành Đơn hàng

⚙️ System Features

  • Pricing Engine: tự điền giá Tier
  • Margin Evaluator: tính realtime từng dòng & tổng
  • PDF Generator: xuất file báo giá chuyên nghiệp

👤 Sale Features

  • Giao diện lập báo giá + Margin realtime
  • Nút "Xuất PDF" gửi Zalo/Email
  • Nút "Chuyển thành Đơn hàng"
  • Danh sách báo giá + trạng thái

👤 Admin Features

  • Danh sách báo giá "Chờ duyệt"
  • Xem chi tiết Margin, giá vốn, chiết khấu
  • Nút Phê duyệt / Từ chối + lý do
📝
Module 5

Đơn Hàng (Orders)

📐 Business Rules

Đơn hàng chỉ được tạo từ Báo giá đã được duyệt. Vòng đời trạng thái đơn hàng:

Draft → Đã xác nhận → Đang chuẩn bị hàng → Đã xuất kho → Hoàn thành

Mỗi chuyển trạng thái ghi nhận: Ai thực hiện, lúc mấy giờ.

⚙️ System Features

  • Sao chép toàn bộ thông tin từ Báo giá sang Đơn hàng khi chuyển đổi

👤 User Features

  • Danh sách đơn hàng: Lọc theo trạng thái, Đại lý, khoảng thời gian
  • Chi tiết đơn hàng: SKU, giá trị, lịch sử trạng thái
📦
Module 6

Nhập Kho & Quản Lý Serial

📐 Business Rules

  • Mỗi đợt nhập hàng tạo một Lô (Batch): Mã lô, ngày nhập, nhà cung cấp, giá vốn nhập
  • Thiết bị có Serial (Inverter, Pin): bắt buộc đăng ký số Serial khi nhập kho
  • Phụ kiện không có Serial: quản lý theo số lượng đơn thuần
  • Trạng thái Serial: In Stock → Shipped → Activated
  • Giá vốn Đích danh: Giá vốn thiết bị = Giá nhập của Lô hàng chứa chính xác Serial đó
  • ✓ Quy trình xuất & nhập kho: Sử dụng số Serial sẵn có của nhà sản xuất. Nhập kho: import Excel/quét. Xuất kho: quét (súng quét/camera PWA) hoặc nhập tay.

⚙️ System Features

  • Serial Validator: Kiểm tra tồn tại, đúng SKU, không trùng phiếu
  • Serial-COGS Mapper: Tra Lô gốc → giá vốn chính xác
  • Traceability Logger: Ghi log toàn bộ vòng đời Serial

👤 User Features (Thủ kho)

  • Nhập Kho: Tạo phiếu → Gắn Lô → Nhập giá vốn → Import Excel Serial
  • Tồn Kho: Xem số lượng theo SKU, phân tách theo Lô
  • Xuất Kho: Gắn phiếu vào Đơn hàng → Chọn dòng chi tiết đơn hàng (Order Line) → Nhập/Quét Serial xuất đi
🛠️
Module 7

Kích Hoạt Bảo Hành (QR Warranty)

📐 Business Rules

  • Ngày hết hạn BH = Ngày kích hoạt + Thời hạn bảo hành chuẩn của SKU
  • Khi kích hoạt: Liên kết Serial ↔ Dòng đơn hàng (Order Line) ↔ Đại lý ↔ End-User
  • Nhân viên Solando quét QR thực địa để xác thực nguồn gốc thiết bị
  • Đại lý gốc đã Inactive → Hệ thống cảnh báo, CSKH Solando xử lý trực tiếp

⚙️ System Features

  • QR Mapping Engine: Serial → Dòng đơn hàng (Order Line) → Đơn hàng gốc → Đại lý
  • Tính toán và lưu ngày hết hạn bảo hành
  • Xác thực Serial thuộc hệ thống Solando

👤 User Features

  • Public QR Form (không cần login): Quét Serial → Điền End-User → Chọn Đại lý → Kích hoạt
  • Kích hoạt nội bộ (Thủ kho/CSKH): Tìm Serial → Điền End-User → Gán Đại lý → Kích hoạt thủ công
  • Tra cứu BH (Public): Nhập Serial → Xem trạng thái, ngày hết hạn

5. Luồng Chính Cần Kiểm Thử (Happy Path)

1 Sale lập báo giá & đơn hàng

• Sale chọn Đại lý
• Hệ thống tự áp giá Tier
• Sale thêm SKU, chỉnh chiết khấu
• Hệ thống tính Margin realtime
⚠ [Nếu Margin thấp] Admin nhận thông báo và phê duyệt
• Sale xuất PDF gửi Đại lý
• Đại lý đồng ý (miệng)
• Sale nhấn "Chuyển thành đơn hàng"
✓ Đơn hàng → "Đang chuẩn bị hàng"

2 Nhập kho & gắn Serial

• Thủ kho tạo phiếu nhập kho
• Tạo mã Lô mới (ngày nhập, giá vốn)
• Import file Excel Serial từ NSX
✓ Hệ thống lưu từng Serial vào Lô, trạng thái "In Stock"

3 Kích hoạt bảo hành QR

• KTV quét QR trên Inverter
• Mở trang kích hoạt (không login)
• Serial tự động điền
• Điền thông tin End-User + Đại lý
• Xác nhận kích hoạt
✓ Hệ thống liên kết: Serial ↔ Đại lý ↔ End-User, tính hạn BH
