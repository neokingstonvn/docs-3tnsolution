1. Mục Tiêu Chi Tiết & Scope Cốt Lõi

Để hoàn thành việc tích hợp trong vòng 14 ngày, đội ngũ kỹ thuật cần chia nhỏ công việc thành 4 giai đoạn độc lập. Hệ thống NestJS sẽ đóng vai trò gateway trung chuyển và xử lý logic gán Serial, còn React (Vite) sẽ đảm nhận hiển thị và tối ưu hóa UX di động cho thủ kho quét mã.

2. Danh Sách Task Triển Khai Chi Tiết

Giai Đoạn 1 Cấu Hình Core Database & API OAuth KiotViet (Ngày 1–4)

  • Task 1.1: [Solando Warranty App] Thiết Lập Database Migration (SQL Setup) PostgreSQL

    Tạo các bảng serial_numbers, warranties, warranty_tickets và thiết lập indexes tối ưu trên PostgreSQL 16.

  • Task 1.2: [Solando Warranty App] Khởi Tạo NestJS Backend Project NestJS

    Scaffold dự án NestJS, cấu hình TypeORM/Prisma, class-validator, và file cấu hình môi trường (.env).

  • Task 1.3: [KiotViet Integration] Client OAuth 2.0 Client Handshake NestJS

    Cài đặt HttpClient, viết module lấy access token từ API KiotViet kèm cơ chế tự động refresh và cache token.

  • Task 1.4: [KiotViet Integration] Service Đồng Bộ Hóa Đơn & SKU NestJS

    Hiện thực API client gọi các endpoint hóa đơn và SKU hàng hóa từ KiotViet API để đối chiếu bán hàng.

Giai Đoạn 2 Phát Triển Giao Diện Thủ Kho & Gán Serial Xuất Kho (Ngày 5–8)

  • Task 2.1: [Solando Warranty App] API Quản Lý Số Serial NestJS

    Viết API endpoints POST /api/serials/assign để gán Serial cho hóa đơn KiotViet và GET /api/serials/invoice/{code}.

  • Task 2.2: [Solando Warranty App] Khởi Tạo Vite + React App React (Vite)

    Setup dự án Frontend, cấu hình Router, Zustand quản lý Client State, và import design system CSS.

  • Task 2.3: [Solando Warranty App] Giao Diện Gán Serial (Warehouse UI) React (Vite)

    Màn hình Thủ kho nhập mã hóa đơn KiotViet để load danh sách SKU, số lượng cần quét gán Serial đích danh.

  • Task 2.4: [Solando Warranty App] Tích Hợp Quét Mã Vạch (Scanner Integration) React (Vite)

    Tích hợp thư viện quét mã html5-qrcode để dùng camera quét nhanh mã Serial, hỗ trợ cơ chế nhập tay khi mã mờ.

Giai Đoạn 3 Phát Triển Cổng Kích Hoạt Bảo Hành QR Public (Ngày 9–11)

  • Task 3.1: [Solando Warranty App] API Kích Hoạt Bảo Hành QR NestJS

    Hiện thực endpoint GET /api/warranties/check/{serial} và POST /api/warranties/activate tính toán ngày hết hạn dựa trên SKU.

  • Task 3.2: [Solando Warranty App] Form Kích Hoạt QR Di Động (Responsive UI) React (Vite)

    Tạo trang form công cộng lấy mã Serial từ query parameters, cho phép điền thông tin End-User để kích hoạt.

Giai Đoạn 4 Phát Triển Tính Năng CSKH & Go-live (Ngày 12–14)

  • Task 4.1: [Solando Warranty App] API Quản Lý Ticket Bảo Hành NestJS

    Tạo endpoints quản lý Ticket bảo trì thiết bị: POST /api/tickets, PATCH /api/tickets/{id} cập nhật trạng thái lỗi.

  • Task 4.2: [Solando Warranty App] Giao Diện CSKH (Tickets Console & Serial Search) React (Vite)

    Xây dựng console quản lý lỗi thiết bị cho CSKH, tìm kiếm lịch sử Serial và gán kỹ thuật xử lý sự cố.

  • Task 4.3: [QA & DevOps] Kiểm Thử E2E Tích Hợp & Deploy VPS DevOps

    Chạy toàn bộ tích hợp KiotViet B2B đơn hàng test, kiểm thử đầu cuối (E2E), đóng gói ứng dụng bằng Docker Compose và deploy lên VPS.

3. Cấu Trúc Database PostgreSQL Chi Tiết (DDL)

Dưới đây là cấu trúc script SQL chính thức phục vụ migration dữ liệu trên database PostgreSQL 16:

-- Kích hoạt extension sinh UUID tự động
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- 1. Bảng lưu trữ Số Serial thiết bị đã bán
CREATE TABLE serial_numbers (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    serial_number VARCHAR(100) UNIQUE NOT NULL,
    product_code VARCHAR(50) NOT NULL, -- SKU hàng hóa KiotViet
    product_name VARCHAR(255) NOT NULL,
    invoice_code VARCHAR(100) NOT NULL, -- Hóa đơn xuất bán trên KiotViet
    sold_date TIMESTAMP NOT NULL, -- Ngày hóa đơn KiotViet
    status VARCHAR(50) DEFAULT 'Sold' CHECK (status IN ('Sold', 'Activated')), 
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 2. Bảng lưu trữ thông tin Kích hoạt Bảo hành
CREATE TABLE warranties (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    serial_number VARCHAR(100) UNIQUE NOT NULL REFERENCES serial_numbers(serial_number) ON DELETE CASCADE,
    customer_name VARCHAR(255) NOT NULL,
    customer_phone VARCHAR(50) NOT NULL,
    installation_address TEXT NOT NULL,
    dealer_code VARCHAR(100), -- Khách hàng mua buôn từ KiotViet
    activation_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expiry_date TIMESTAMP NOT NULL, -- Hạn bảo hành thực tế tính toán
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 3. Bảng lưu trữ nhật ký Yêu cầu Bảo hành (Tickets)
CREATE TABLE warranty_tickets (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    serial_number VARCHAR(100) NOT NULL REFERENCES serial_numbers(serial_number) ON DELETE CASCADE,
    status VARCHAR(50) DEFAULT 'Pending' CHECK (status IN ('Pending', 'In Progress', 'Resolved', 'Rejected')),
    problem_description TEXT NOT NULL,
    cskh_notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Tạo Index để tối ưu hiệu năng tìm kiếm nhanh
CREATE INDEX idx_serials_number ON serial_numbers(serial_number);
CREATE INDEX idx_serials_invoice ON serial_numbers(invoice_code);
CREATE INDEX idx_warranties_serial ON warranties(serial_number);
CREATE INDEX idx_tickets_serial ON warranty_tickets(serial_number);

4. Tích Hợp API KiotViet (Access & Payload Map)

OAuth 2.0 Auth Handshake

NestJS gọi API token KiotViet qua giao thức HTTP POST:

POST https://id.kiotviet.vn/connect/token Headers: Content-Type: application/x-www-form-urlencoded Body: scopes=PublicAPI.Access&grant_type=client_credentials&client_id=CLIENT_ID&client_secret=CLIENT_SECRET
GET /invoices/code/{code} Response Schema

Cấu trúc JSON nhận về từ KiotViet API để NestJS lấy danh sách SKU đối chiếu số lượng quét gán Serial:

{ "id": 10001452, "code": "HD000145", "purchaseDate": "2026-06-27T15:00:00", "customerCode": "DL0009", "customerName": "Đại Lý Năng Lượng Xanh", "invoiceDetails": [ { "productCode": "INVERTER-5KW", "productName": "Biến tần Solando 5kW", "quantity": 2, "price": 15000000 } ] }

5. Kế Hoạch Kiểm Thử & Chạy Thử (QA Plan)

Kiểm Thử Tự Động
  • OAuth mock tests: Giả lập kết nối mạng KiotViet timeout, kiểm tra cơ chế retry.
  • Serial Validator unit tests: Check validate không cho phép lưu Serial rỗng hoặc sai số lượng so với hóa đơn gốc.
UAT Bằng Tay
  • Quét gán kho: Đóng vai thủ kho sử dụng thiết bị quét thử mã máy dán tem QR để kiểm tra tính mượt mà của camera.
  • Kích hoạt QR: Dùng điện thoại quét QR link, điền thông tin End-User, kiểm tra bảng database ghi nhận chính xác.
